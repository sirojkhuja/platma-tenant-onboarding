Build a NestJS service that handles tenant onboarding. When a POST /tenants request is received with a tenant name and admin email, the service should: create a record in PostgreSQL (using TypeORM or Prisma), register a client and user in Keycloak via its Admin REST API, and generate a Kubernetes manifest (Deployment + Service YAML) for a hypothetical per-tenant Node-RED instance — writing it to disk or returning it in the response. A DELETE /tenants/:id endpoint should reverse the process: mark the tenant as inactive in PG, disable the Keycloak client, and produce a corresponding delete manifest. No need to actually apply the manifests to a live cluster.
Use @nestjs/config for all connection strings and secrets. Include a docker-compose.yml that spins up PostgreSQL and Keycloak locally for development. Write a few integration tests using NestJS testing utilities (@nestjs/testing) that walk through the create → verify → delete lifecycle. Be prepared to discuss how you'd extend this to actually apply manifests via the K8s API, handle partial failures (e.g., Keycloak succeeds but PG fails — compensating transactions, sagas), and what changes you'd make for production readiness.
